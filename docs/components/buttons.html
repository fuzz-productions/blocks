--- layout: componentSection title: Buttons status: Design review version: v.0 definition: Buttons perform actions, like saving, cancelling or sending, or call users to those actions. They should not be used for navigation (“Back” or “Home”), unless the button’s action has navigational implications (“Save and close”). usage: A screen can have lots of buttons, but only two of them can be the primary button style. The rest must be secondary. Up to three buttons can be grouped together in a row or columns, but only the last one in the group can be primary. Text on button labels should be written in sentence case: capitalize only the first word and any proper nouns. Button text should always start with a verb that describes the button’s action, and should be no longer than three words. --- {% capture primary_button_example %} {% endcapture %} {% capture primary_button_hover_example %} {% endcapture %} {% capture primary_button_focus_example %} {% endcapture %} {% capture primary_button_disabled_example %} {% endcapture %} {% capture secondary_button_example %} {% endcapture %} {% capture secondary_button_hover_example %} {% endcapture %} {% capture secondary_button_focus_example %} {% endcapture %} {% capture secondary_button_disabled_example %} {% endcapture %} {% capture primary_button_icon_example %} {% endcapture %} {% capture secondary_button_icon_example %} {% endcapture %} {% capture primary_button_icon_right_example %} {% endcapture %} {% capture secondary_button_icon_right_example %} {% endcapture %} {% capture h_button_group_example %}
